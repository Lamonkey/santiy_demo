# Headless CMS Content Platform


- Built a personal portfolio as the project vehicle to practice Sanity best practices end-to-end — not just following tutorials but deliberately applying real-world Studio UX patterns
- Designed the Studio editing experience with content editor UX in mind — used conditional fields to hide form inputs until they became relevant, reducing cognitive load for the editor
- Built a customized content editing interface using Sanity's React hooks, going beyond the default Studio UI to create purpose-built editing components
- Practiced content and schema migration using Sanity's migration tools — simulating how you handle content model changes in a real production environment
- Used Sanity blueprints for post-processing through function calling — built an AI-assisted blog workflow where clicking unpublish triggers a blueprint that calls an AI function to review the draft and return feedback before the post goes live
- Building a headless CMS-powered content platform in Sanity and Next.js that enables non-technical users to create and manage dynamic content without code
- Designed a flexible Sanity schema to model a fully customizable landing page composed of modular sections (hero, features, testimonials), a structured blog system with authors and categories, and detailed case study pages
- Writing GROQ queries to fetch and surface structured content across the site, with typed responses in TypeScript for reliable data handling on the frontend
- Implemented Portable Text rendering with custom block components in Next.js for rich, structured editorial content
- Customized Sanity Studio with a custom desk structure and input components to streamline the content editing experience for non-technical users
- Built search and filtering, SEO optimization (meta tags, structured data), and an AI-assisted feature to generate summaries and metadata directly within the CMS
- Integrated Sanity's content pipeline with Next.js SSG and SSR patterns for fast, dynamic frontend rendering
